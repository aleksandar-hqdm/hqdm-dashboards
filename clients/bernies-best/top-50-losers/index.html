Why this matters — the Ideal Sitemap connection

The Ideal Sitemap workflow maps a client's current content state vs. ideal state for a topical cluster. Output is an Excel sheet keyed by main_keyword with: cluster KWs, search volume, existing URLs, current rankings, on-page metrics (wordcount, schema, images, internal/external link counts), and rule-based suggested_actions: solve_cannibalization, manage_redirects, upgrade_main_article, consolidate_learn_more, write_new_article, write_new_page, review_manually.

The Bernie's MVP is the Perfect Poop cluster (digestive / gut / stool — 39 main_keyword clusters). The connection: the top-50 losers below are joined against the sitemap actions. Where a URL appears in a sitemap cluster, the rightmost column shows the action recommendation. This is how loss inventory becomes execution: each loser carries a planned action keyed off the cluster-level analysis, not an ad-hoc per-URL hypothesis. Same workflow now reusable for Bernie's 4 remaining SKU clusters and every future HQDM client.

Action distribution across the 15 mapped losers: redirect ×12 cannibalization ×12 upgrade ×5 consolidate LM ×5 review ×1. Dominant pattern is redirect-chain + cannibalization (the 5×5 Learn More problem documented in store-gap.html), not pure upgrade.

What the action recommendations mean

upgrade

upgrade_main_article — A single existing Bernie's URL ranks for the cluster's main_keyword but content is thin, missing modern on-page signals, or below S&T-pattern depth. Apply the S&T-style restructure (numbered H3 enumeration, expand to 2,100-3,500w, FAQ + Final Thoughts, body-cited vet credentials, 5-8 product mentions). Detailed pattern: st-pattern.

cannibalization

solve_cannibalization — 2+ Bernie's URLs rank for queries in the same cluster (blog + Learn More + Knowledge Base all hitting the same SERP). Apply pick one canonical winner, 301-redirect the rest. Common pattern: 5 PDPs × 5 "Learn More" pages on www.bernies.com cannibalize the corresponding store PDPs.

redirect

manage_redirects — Existing 301 redirects in the cluster route to a destination on a different sub-topic (URL hygiene gap). Apply either un-redirect (republish a properly-scoped article) or retitle/restructure the destination to legitimately cover both topics. Example: throwing-up-clear-liquid → throwing-up-white-foam (different sub-topics).

consolidate LM

consolidate_learn_more — A Perfect Poop "Learn More" subpage exists on www.bernies.com that mirrors content on the store PDP. Apply merge the Learn More content into the PDP (or related blog) and 301 the Learn More URL. Lifts canonical signal on the PDP.

new article / page

write_new_article / write_new_page — Cluster has substantial SV / multiple competing URLs but Bernie's has no existing URL. Apply greenfield content. Lower priority (most upside is hygiene + cornerstone upgrades, NOT new articles).

review

review_manually — Signals are ambiguous — cluster has multiple existing URLs at moderate ranks AND incomplete cluster volume data, or partial cannibalization pattern. Apply senior SEO judgment call. Aleksandar reviews before action.
