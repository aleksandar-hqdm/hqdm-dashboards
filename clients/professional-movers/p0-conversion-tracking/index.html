PM's GA4 event stack vs the AQMS reference

AQMS is HQDM's reference moving-account: four lead-intent events firing in GA4, all marked as conversion events, composite leads metric reported monthly. PM has form_submit firing — the rest of the stack needs deployment.

M1 action sequence

  1. AN-01 — Mark form_submit as a conversion event. GA4 → Admin → Events → find form_submit → toggle "Mark as conversion." All reports populate the conversions column from this point forward. Not retroactive. 5 min, HQDM-side, no client auth.
  2. AN-02 — Deploy click_phone_number + gmb_click events via GTM. Mirror the AQMS event config: tel: link click trigger → click_phone_number GA4 event; GBP-tagged link click trigger → gmb_click GA4 event. Mark both as conversion events in GA4. HQDM-side, no client auth, ~2-day implementation.
  3. AN-03 — Verify GBP UTM is firing. Open GA4 Realtime → click GBP "Website" link from incognito → confirm session shows source: gmb, medium: click. Re-verify after GBP optimization (Lever A GBP-03..09) and after Detroit Woodward + Rochester pins go live so the UTM is consistent across all three pins.
  4. AN-04 — Build Q3 baseline. Once the full event stack is live, pull 30/60/90d baseline per channel for the composite leads metric (sum of form_submit + click_phone_number + gmb_click). Update dashboard data.json.
  5. AN-05 — First monthly report with composite leads metric. Lead with the divergence chart (sessions + composite leads, channel-split) + per-event breakdowns where relevant.
  6. AN-06 — Q3 → Q4 quarterly comparison frame. Establish the reporting cadence carrying forward.

Why this unblocks everything else

Every other lever — Map Presence (A), Page Surgery (B), Content (D) — is measured by lead lift, not just impression lift. The Lever A case (claim Detroit Woodward + Rochester GBPs) rests on the data point that GBP→site click-throughs convert at ~2× organic. Adding the full lead-event stack lets the Q3 report show channel-specific lead impact per workstream and isolate which lever moved the needle. A 5-min property-settings flag plus a 2-day GTM rollout protects 90 days of measurement integrity.
