MINI-REPORT · P0 PREREQUISITE

Conversion tracking restoration — 5-minute GA4 fix + CallRail deployment

Until form_submit is flagged + CallRail captures phone-attribution, every conversion KPI is a derived count, not a GA4-reported number.

What's broken vs what's working

Working
  • · form_submit event IS firing — ~190 events/mo across channels (DataLayer push from Gravity Forms confirmed via GA4 DebugView)
  • · GBP UTM (?utm_source=gmb&utm_medium=click) IS firing into GA4 — captured as 149 sess/mo (30d)
  • · Channel grouping resolves correctly into Direct / Organic / Referral
  • · Property 385147451 has API access via info@hqdm.io OAuth
Broken
  • · form_submit NOT flagged as a conversion event in property settings → GA4 reports "0 conversions / 65K sessions / 901 pages / 365d"
  • · Phone calls (the dominant lead type for movers) not tracked at all — CallRail not deployed
  • · No GA4 Audience defined for converters → can't segment retargeting
  • · Reports built on GA4 conversion column will show 0 across the board

5-min GA4 fix — step-by-step

  1. AN-01: Mark form_submit as a conversion event. GA4 → Admin → Events → find form_submit → toggle "Mark as conversion." From this moment forward, all reports populate the conversions column. Backfill is NOT retroactive — only forward.
  2. AN-02 (parallel): Deploy CallRail for phone-call attribution. Owner authorization required (OCI Ask 3). CallRail dynamic-number-insertion script + GA4 event mapping for call_completed. Set up CallRail → Integrations → GA4 → paste Measurement ID + API secret. Then mark call_completed as conversion event in GA4 (mirrors AN-01 for the second conversion type).
  3. AN-03: Verify GBP UTM is firing. Open GA4 Realtime → click GBP "Website" link from incognito → confirm session shows source: gmb, medium: click in Realtime. Already firing per our last 30d data; re-verify after GBP optimization (Lever A GBP-03..09) doesn't break the UTM.
  4. AN-04: Build Q3 baseline. Using GA4's now-active conversion column + CallRail data, pull 30/60/90d baseline per channel. Update data.json trend block with real conversion numbers (replacing derived form_submit counts).
  5. AN-05: First monthly report. Format per the GA4↔GSC reconciliation mini-report. Lead with the trend chart, channel split, GBP highlight.
  6. AN-06: Q3 → Q4 quarterly comparison frame. Establish the reporting cadence so the Q4 report can show "tracking restored M1; here's what M1-M3 actually converted."

Owner asks required to start

  • · CallRail authorization + budget — typical ~$45-90/mo for 5-10 numbers. PM operates 3 offices + GBP, so 4-5 numbers minimum.
  • · Owner contact list of current phone-attribution sources — to validate that CallRail numbers cover all "incoming call" channels (forms, GBP, paid, referral).
  • · Confirmation of conversion source priorities — call > form > schedule-video-estimate? Drives the conversion-event weighting in GA4.

Why this unblocks everything else

Every other lever — Map Presence (A), Page Surgery (B), Content (D) — is measured by conversion lift, not just impression lift. Without conversion data flagged in GA4, every M1/M2/M3 review will be qualitative. With AN-01 + AN-02 done in M1 week 1: the Q3 report can show channel-specific conversion impact for each lever, isolate which workstreams moved the needle, and justify continued investment per channel. 5-min fix + 2-day CallRail rollout protects 90 days of measurement integrity.
