MINI-REPORT · P0 PREREQUISITE

Three gaps in the lead measurement stack — form_submit flag + CallRail + GBP Insights

PM dashboards currently report ONE slice of lead generation — web-form submits on the website — and call it "conversions." Three measurement gaps stack on top of each other: the GA4 form_submit flag isn't set, CallRail isn't deployed for phone calls (the dominant mover-lead path), and GBP Insights has never been pulled for listing-side actions. M1 closes all three.

What we measure today vs what we don't

A "lead" for PM has at least three independent surfaces. We currently see one. This is why the dashboard headlines say "GBP CR 17.5%" and "highest-converting channel" — those are true on the form-submit slice but are not statements about the channel's overall lead contribution.

Layer 1 — Web-form submits (visible)

~190 events/mo across all channels. Source: GA4 form_submit event (Gravity Forms DataLayer push, confirmed via DebugView). Per-channel breakdown: Direct 36–111, Organic 28–71, GBP→site 18–24 (37 est. May 2026), LLM 0–4. Channel grouping resolves correctly via UTMs.

Gap: the event fires but is NOT flagged as a conversion event in GA4 property settings → GA4 native reports show "0 conversions." 5-min HQDM-side fix (AN-01).

Layer 2 — Phone-call leads (invisible)

Volume unknown. Industry pattern: 2–5× the web-form number for local movers. PM publishes three phone numbers (Walled Lake (248) 926-9999 / Rochester (586) 371-7777 / Detroit (313) 964-4444). None have dynamic-number-insertion tracking; none are routed to GA4 events. We don't know which call came from which Google source — or whether the GBP listing's "Call" button drove the call vs an organic-search visit to /about-us/contact/.

Gap: requires CallRail deployment (~$45–90/mo). Owner authorization needed (OCI Ask 3) → AN-02.

Layer 3 — GBP listing actions (un-pulled)

Volume unknown — Insights never pulled. Every action taken on the GBP listing itself — clicking "Call," tapping "Directions," sending a message, viewing photos, clicking "Website" — is logged by GBP and exposed via the GBP Performance API. None of this has been pulled for PM. The "Website" subset shows up in GA4 (Layer 1) as utm_source=gmb, but the calls + directions + messages have never been counted.

Gap: requires monthly GBP Insights API pull (HQDM-side, no client auth) → AN-07 (new).

Why this matters for client comms: Statements like "GBP is the highest-converting channel" or "GBP CR ~17.5% vs organic 9.2%" are technically claims about Layer 1 only — the form-submit rate of website visitors who came in via the GBP "Website" button. They are true and meaningful, but they're not statements about total GBP lead generation. The full lead picture stays open until all three layers are instrumented. Recommend client-facing copy uses "form submissions" rather than "conversions" or "leads" until Layers 2 + 3 are live.

M1 action sequence — three gaps, three closes

  1. AN-01 — Close Gap 1 (web form-submit flag). GA4 → Admin → Events → find form_submit → toggle "Mark as conversion." From this moment forward, all reports populate the conversions column with web form-submit events. Backfill is NOT retroactive — only forward. 5 min, HQDM-side, no client auth.
  2. AN-02 — Close Gap 2 (phone-call attribution). Deploy CallRail dynamic-number-insertion for all three PM numbers + GA4 event mapping for call_completed. CallRail → Integrations → GA4 → paste Measurement ID + API secret. Then mark call_completed as a conversion event in GA4. Owner authorization required (OCI Ask 3, ~$45–90/mo). This is the gap that matters most for understanding total lead volume — calls are typically the dominant mover-lead path.
  3. AN-07 — Close Gap 3 (GBP listing-side actions). Pull GBP Insights / Performance API monthly for the Walled Lake pin (and Detroit Woodward + Rochester pins after Lever A claims). Capture: searches that surfaced the listing (direct vs discovery), profile views, "Website" clicks, "Call" clicks, "Direction" requests, photo views, messages received. Build a monthly dashboard row alongside form-submits + calls. No client auth needed — runs on existing HQDM GBP manager access.
  4. AN-03: Verify GBP UTM is firing. Open GA4 Realtime → click GBP "Website" link from incognito → confirm session shows source: gmb, medium: click. Already firing per our last 30d data; re-verify after GBP optimization (Lever A GBP-03..09) and after Detroit Woodward + Rochester pins go live to ensure UTM is consistent across all three pins.
  5. AN-04: Build Q3 baseline. Using AN-01 + AN-02 + AN-07 data, pull 30/60/90d baseline per channel + per pin. Three-layer view: web form-submits, phone calls (per channel + per pin), GBP listing actions (per pin). Update dashboard data.json with the complete view.
  6. AN-05: First monthly report — three-layer lead view. Lead with the divergence chart (sessions + web form-submits) + a NEW combined-lead chart (form + call + listing-action stacked per channel/pin). Resolves the "is GBP the highest-converting channel" question on actual lead volume rather than the form-only slice.
  7. AN-06: Q3 → Q4 quarterly comparison frame. Establish the reporting cadence so the Q4 report can show "measurement gaps closed M1; here's what M1-M3 actually drove across all three lead layers."

Owner asks required to start

  • · CallRail authorization + budget — typical ~$45-90/mo for 5-10 numbers. PM operates 3 offices + GBP, so 4-5 numbers minimum.
  • · Owner contact list of current phone-attribution sources — to validate that CallRail numbers cover all "incoming call" channels (forms, GBP, paid, referral).
  • · Confirmation of conversion source priorities — call > form > schedule-video-estimate? Drives the conversion-event weighting in GA4.

Why this unblocks everything else

Every other lever — Map Presence (A), Page Surgery (B), Content (D) — is measured by lead lift, not just impression lift. Today we measure only one of three possible lead types. The Lever A case rests on the premise that "GBP is the highest-converting channel" — true on the form-submit slice, unknown on the call + listing-action slices. If CallRail + GBP Insights confirm that GBP also leads on calls and listing-side conversions, the Lever A ROI argument gets ~3× stronger. If they don't, we re-balance the lever weighting. Either way, the team stops making partial-view claims to clients.

With AN-01 + AN-02 + AN-07 done in M1 week 1: the Q3 report can show channel-specific lead impact across all three layers, isolate which workstreams moved the needle on which surface, and justify continued investment per channel. 5-min flag + 2-day CallRail rollout + 1-day GBP Insights pull protects 90 days of measurement integrity.
